# 映链新增需求补充（日志中文化 + Emby 统计区）

## 1. 目标
本次新增需求有两个目标：

1. 提升日志页可读性：让非技术用户也能快速看懂日志状态与结果。
2. 提升首页信息价值：在首页下方增加 Emby 媒体库统计区，让用户一眼看到当前媒体库规模与最近新增情况。

本次改动默认沿用当前「映链」浅色高级感 UI 风格，不改变整体视觉方向。

---

## 2. 新增功能点说明

### 功能 A：运行日志中文化

#### A.1 日志级别中文化
前端日志筛选和日志项显示统一改为中文：
- `INFO` → `信息`
- `WARN` → `警告`
- `ERROR` → `错误`

#### A.2 日志内容中文可读化
对系统内可控的日志文案，优先改为中文表达，例如：
- `Scan complete` → `扫描完成`
- `Generation started` → `生成任务已开始`
- `Cleanup started` → `清理任务已开始`
- `Config set` → `配置已更新`

如果日志里包含技术字段、路径、异常栈等原始信息：
- 保留原始技术内容
- 但在展示时增加中文语义包装，避免整条日志都是生硬英文

示例：
- `Config set: source_dir = /data/clouddrive/gdrive`
  → `配置已更新：媒体源目录 = /data/clouddrive/gdrive`

---

### 功能 B：首页新增 Emby 统计区

在首页主统计卡片下方新增一组 Emby 媒体库统计展示，至少包含：
- 电影数量
- 电视剧数量
- 总数量
- 最近新增数量

该区域用于体现：
- 当前 Emby 媒体库规模
- STRM 生成结果对 Emby 的承载效果
- 最近是否有新增内容进入库中

---

## 3. 首页展示建议

### 3.1 位置
放在首页主统计卡片区下方，任务状态卡片上方或下方均可，但必须形成独立模块。

推荐结构：
1. 第一行：原有映链任务统计卡
2. 第二行：**Emby 媒体库统计区**
3. 第三行：最近任务状态 / 空状态 / 输出信息

### 3.2 展示方式
建议采用 **4 张并列统计卡片**：
- 电影数量
- 电视剧数量
- 总数量
- 最近新增数量

每张卡片包含：
- 小图标
- 中文标题
- 大数字
- 一行辅助说明

### 3.3 卡片风格建议
沿用当前浅色高质感设计：
- 白底或浅渐变底
- 彩色图标圆底
- 大数字居中或偏左展示
- 辅助说明文字浅灰
- hover 微浮起

建议配色：
- 电影数量：蓝色系
- 电视剧数量：紫色系
- 总数量：绿色系
- 最近新增：橙色系

---

## 4. 数据字段定义

前端展示字段建议统一为：

```json
{
  "movie_count": 0,
  "series_count": 0,
  "total_count": 0,
  "recent_added_count": 0,
  "recent_range_days": 7,
  "last_updated_at": "2026-04-11T10:30:00Z"
}
```

### 字段说明
- `movie_count`：Emby 中电影总数
- `series_count`：Emby 中电视剧总数
- `total_count`：Emby 总条目数（电影 + 剧集；如果后端定义更广，需明确口径）
- `recent_added_count`：最近新增数量
- `recent_range_days`：最近新增统计口径，建议默认 7 天
- `last_updated_at`：最近一次成功拉取 Emby 数据时间

---

## 5. 交互与空状态要求

### 5.1 加载中状态
- 卡片显示 skeleton 或浅灰占位
- 不要闪烁
- 区块标题保留

### 5.2 空状态
如果 Emby 尚未配置、接口不可用或未返回数据：
- 区块仍保留
- 不显示报错堆栈
- 用友好中文提示：
  - `尚未连接 Emby，暂无媒体库统计数据`
  - `请先完成 Emby 连接配置后查看统计`

### 5.3 错误状态
如果 Emby 接口请求失败：
- 页面不应整体报错
- 仅该区块展示错误提示
- 提示建议：
  - `Emby 统计获取失败，请检查服务地址或 API Key`

### 5.4 更新时间显示
建议在区块右上角或底部显示：
- `最近更新：2 分钟前`
- 或 `更新时间：2026-04-11 10:30`

## 6. 验收标准

### 日志中文化验收
- [ ] 日志页筛选按钮全部中文
- [ ] 日志级别标签全部中文
- [ ] 系统可控日志文案尽量中文化
- [ ] 原始技术信息保留，但整体可读性明显提升

### Emby 统计区验收
- [ ] 首页新增独立 Emby 统计模块
- [ ] 至少展示电影数量、电视剧数量、总数量、最近新增数量
- [ ] 数据卡样式与当前 UI 风格一致
- [ ] 加载中、空状态、错误状态都有明确中文反馈
- [ ] 页面在无 Emby 数据时不崩溃
- [ ] 统计区支持刷新后稳定展示

## 7. 需要开发确认的技术接口点

这是开发前必须确认的点：

### 7.1 Emby 数据来源
需要明确以下其一：
1. 直接读取 **Emby API**
2. 读取现有中间服务/脚本接口
3. 读取本地数据库/缓存结果

### 7.2 最近新增数量口径
建议默认定义为：
- 最近 7 天新增入库的媒体数量

但开发需确认：
- Emby API 是否能直接返回该字段
- 是否需要按 `DateCreated` / `PremiereDate` / `DateLastRefreshed` 计算

### 7.3 总数量口径
需确认：
- 是否 = 电影 + 电视剧
- 还是包含季、集、音乐、综艺等所有媒体条目

默认产品建议：
- 首页先展示 `电影 + 电视剧` 汇总后的总数量，避免口径混乱

### 7.4 配置入口
若需要调用 Emby API，则后端需补充配置项：
- Emby 服务地址
- API Key
- 用户 ID（若接口需要）

若本轮不做配置页，可先在后端环境变量中配置，前端只负责展示。
