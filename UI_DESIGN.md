# 映链 UI 设计规范

**英文副标题：** STRM Media Bridge

## 1. 设计理念
本产品定位为面向个人媒体库管理的 STRM 生成工具，界面风格应从“工具感”升级为“产品感”。整体采用**浅色主界面 + 深蓝侧边栏 + 彩色数据卡片**的组合，以清晰的信息层级、柔和但有辨识度的配色、轻量阴影和简洁图表，营造专业、舒适、现代的桌面控制台体验。

目标关键词：**清爽、专业、可视化、数据感、轻盈、精致**。

## 6. 卡片组件设计
- **圆角：** 20px
- **阴影：** `0 10px 30px rgba(15, 23, 42, 0.08)`
- **边框：** `1px solid #E2E8F0`
- **背景：** 白色或浅渐变
- **Hover：** 轻微上浮 `translateY(-2px)` + 阴影增强

### 卡片类型
1. **数据统计卡**：大数字 + 标签 + 小图标 + 彩色背景
2. **任务状态卡**：状态徽标 + 当前进度 + 进度条 + 说明
3. **日志卡**：浅灰背景代码块风格，支持滚动
4. **配置卡**：分组表单，区块清晰

## 9. 页面结构建议

## 9.1 总览页
顶部：
- 产品名「映链」
- 英文副标题 `STRM Media Bridge`
- 一句描述：把媒体目录优雅地转换为可播放的 STRM 入口

主体：
1. 第一行 4 张统计卡
   - 电影数量
   - 剧集数量
   - STRM 总数
   - 当前任务进度
2. 第二行
   - 左侧：任务趋势 / 最近任务状态（图表或时间线）
   - 右侧：任务状态总览（idle/running/success/failed）
3. 第三行
   - 最近日志摘要
   - 最近生成路径 / 输出目录信息

## 9.2 配置页
- 分区式表单卡片：源目录、输出目录、命名规则、认证配置
- 每组配置有标题、副说明、输入框/选择器
- 底部固定保存按钮

## 9.3 任务页
- 当前任务状态卡
- 扫描 / 生成 / 清理 操作区
- 进度条 + 任务详情 + 实时状态文本
- 历史任务表格：时间、类型、状态、输出量

## 9.4 日志页
- 支持浅底代码面板
- 过滤：全部 / 错误 / 成功 / 运行中
- 每条日志带状态标签和时间

## 11. 项目名称建议
### 首选
- **中文名：映链**
- **英文副标题：STRM Media Bridge**

命名含义：
- “映”对应影视、媒体
- “链”对应链接、桥接、STRM 的路径映射属性
- 简洁、有工具感，也有科技产品感

### 备选
1. **映桥** / Media Link Bridge
2. **源映** / Stream Source Mapper
3. **媒引** / STRM Route Engine

## 12. 产品经理补充要求（给前端开发）
1. 所有页面**全面中文化**，不保留英文导航词。
2. 所有关键数据要用**可视化更强的卡片、颜色、进度条、徽标**表达，不要只用纯文本。
3. Dashboard 不再只是静态数字，应体现：
   - 当前运行态
   - 历史任务摘要
   - 目录状态
   - 结果分布
4. 页面风格从“开发者工具”升级为“成熟产品后台”。
5. 默认以**浅色方案**为主，不再用单调深色控制台风格。
6. 即使当前数据为空，也要通过优雅的空状态设计保持高级感。
